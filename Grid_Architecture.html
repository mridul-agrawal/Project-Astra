Tier 1 — Authored Data ScriptableObjects tuned in the Inspector — the "what" of a map

MapDataScriptableObject
One battle map's authored data: dimensions, per-layer tile-ID grid, tileset references, unit start positions, event triggers.
uses MapLayer · holds TilesetDefinition[]
TilesetDefinitionScriptableObject
Resolves a tile ID to both a renderable sprite and its TerrainType, so visuals and gameplay data cannot drift apart.
classifies via TerrainType · yields TileBase
TerrainStatTableScriptableObject
Lookup of per-terrain movement cost, DEF/AVO cover bonuses, and heal-per-turn. One row per TerrainType.
keyed by TerrainType · calls Core/Pathfinding (external)
SyncedAnimatedTileTileBase asset
A self-animating tile (water, lava) pinned to global time so all instances share a frame. Slotted into a TilesetDefinition's tile list.
no code deps · is one of the TileBase assets
↓read at runtime · LoadMap(MapData)

Tier 2 — Runtime MonoBehaviours that live in the scene — the "doing"

MapBootstrapperMonoBehaviour
Temporary entry point: on Start() it hands a MapData asset to the MapRenderer. To be replaced when scene/chapter loading exists.
→ MapRenderer.LoadMap · holds MapData
MapRendererMonoBehaviour
The workhorse and the folder's outward API. Stamps MapData onto one Unity Tilemap per layer. SwapTile() handles destructibles and fires OnTileSwapped. GetTerrainType(x,y) answers combat and movement.
reads MapData · TilesetDefinition · MapLayer · TerrainType · emits OnTileSwapped (external) · exposes GetTerrainType (external)

Standalone Utility no coupling to anything else in Grid

OverlaySpriteFactorystatic class
Generates and caches the 16×16 hover-highlight sprite at runtime (1px border + diagonal gradient). Pure sprite math, consumed by the grid cursor / overlay system elsewhere.
zero Grid dependencies · consumed by Cursor / Overlay (external)
