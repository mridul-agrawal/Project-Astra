Workflow 01

Patch-by-patch with seed lock

The minimum-viable improvement to your current process. Stay with Edit Image (Pro), but lock the random seed and overlap your patches so each generation sees its neighbour's edge as context.

Tool
Edit Image (Pro)
Patch size
256×256 (×4)
Tier
Pro
Effort
Low
Procedure
01

Open the 512×512 map in Pixelorama

Either continue from an existing project (.pxo) or import an existing PNG via File → Open. Confirm the canvas dimensions in the top bar read [512×512].

02

Plan four overlapping quadrants

Instead of dividing the canvas into clean 256×256 quadrants at (0,0), (256,0), (0,256), (256,256), plan an overlap zone of 64-128 pixels between adjacent patches. Each subsequent patch will see already-generated content as edge context.

Patch 1: 0,0 → 256,256 Patch 2: 192,0 → 448,256 overlap: 64px
03

Make the first selection — and commit it

Use the Rectangular Selection tool. Drag a rough box, then refine using the X / Y / Width / Height fields in the Left Tool Options panel. Press Enter (or click the green checkmark) to commit — this is critical, otherwise the selection stays in transformation mode and the field edits will move the content under it instead of the boundary.

04

Open Edit Image (Pro) and lock the seed

From the PixelLab menu select Edit Image (Pro). Open Advanced options and find the Seed field. Pick any integer (e.g., 12345) and write it down — you will reuse this exact value for every patch in this map.

Seed: documented option 256×256 max: from your testing
05

Write a precise instruction and generate

Use action verbs: change, add, remove, replace. The docs explicitly recommend this phrasing. Example: "replace with detailed top-down grassland tiles, paths, scattered trees". Click Generate.

06

Move to the next patch — keep the same seed

Make a new selection that overlaps the previous patch by 64-128px. Re-open Edit Image (Pro). Confirm the seed is still 12345. Use the same instruction text. Generate.

07

Repeat for the remaining quadrants

Work in a consistent traversal order — e.g., top-left → top-right → bottom-right → bottom-left — so each new patch always borders an already-generated one.

08

Manual cleanup at seams

Even with seed-locking + overlap, expect minor discontinuities at patch boundaries. Use the pencil and color-picker tools to manually blend roads, terrain transitions, and water edges where patches meet.

Caveats & uncertainties
  • The 256×256 size limit on Edit Image (Pro) is from your direct observation, not from the docs page (which I could not fetch).
  • Seed-locking reduces style drift but does not eliminate it — diffusion models can still produce variation given different surrounding contexts.
  • The official docs for the standard Edit Image tool list these options: Instruction, Guidance weight, Target Palette, Remove background, Output method, Seed. The Pro variant likely shares these but I have not verified.
Workflow 02

Anchor patch + init image chain

Generate one patch you love. Then use that patch as an init image for every subsequent generation, anchoring style across the entire map. Switches to a tool with verified init-image support.

Tool
Create M-XL image (pixflux)
Max patch size
200 / 320 / 400
Tier
Free / T1 / T2+
Effort
Medium
Procedure
01

Generate one anchor patch

Create or pick the single best-looking 256×256 region of your map. This becomes your style anchor. If starting fresh, generate it first using Create M-XL (pixflux) with no init image — just a strong description.

Init image: documented option Free tier max: 200×200 Tier 1 max: 320×320 Tier 2+ max: 400×400
02

Save the anchor as a separate file

Export the anchor patch as a standalone PNG via File → Export. You will load this PNG as the init image input for every subsequent generation.

03

Set up the next patch's selection

Make a selection in the unfilled area of your map at the size you want to generate. Commit the selection (Enter / green check).

04

Open Create M-XL (pixflux) and load the init image

From the PixelLab menu select Create M-XL image (new). Find the Init image option. When enabled, a small thumbnail of the currently selected init image appears under the dropdown — set this to your saved anchor PNG.

05

Set init image strength to 600-900

The docs are explicit on these ranges:
300-400 — rough shape/color guidance
400-600 — variations on the existing image
600-900 — detailed style enforcement
For style coherence across patches, you want the higher band.

Init strength values: from docs
06

Lock seed and write the description

Use a fixed seed across all patches. Write a description appropriate to this patch's content (e.g., one patch is "forest with stream," another is "rocky hills with cave entrance") — but the same style anchor and seed enforce visual consistency.

07

Generate and review

Compare against your anchor — colors, line weights, dithering pattern, and outline thickness should now match much more closely than freeform generations. Iterate strength up or down by ±100 if results drift.

08

Composite patches in Pixelorama

Position each generated patch at the appropriate location on the 512×512 master canvas. Use the Move tool with selection to nudge alignment. Manual seam cleanup as in Workflow 01.

Caveats & uncertainties
  • Workflow uses Create M-XL (pixflux) instead of Edit Image (Pro) because the pixflux tool's init-image support is documented; Edit Image (Pro) init-image support is not verified in docs I could access.
  • Init image strength is most reliable around 700; values above 850 may produce results too similar to the anchor (and below 500, too divergent).
  • Pixflux size limits depend on your subscription tier — confirm yours before planning patch dimensions.
Workflow 03

Single-pass with Create S-XL (Pro)

The cleanest fix for patch coherence: generate the entire 512×512 map in one diffusion pass. Pro tier required, but eliminates the patch-stitching problem entirely.

Tool
Create S-XL image (Pro)
Max size
512×512 (or 688×384)
Tier
Pro
Cost
40 generations
Procedure
01

Confirm Pro tier subscription

Create S-XL is a Pro-only tool. The cost scales by canvas dimension — at 512×512 each generation costs 40 generations from your quota. Plan attempts accordingly.

Pro tier required: documented ≤256px: 20 generations 257-341px: 25 generations 342-512px: 40 generations
02

Create or open a 512×512 canvas

The tool supports any dimension from 16 to 512 per side for square canvases (or larger non-square aspect ratios — up to ~688×384 for 16:9). Confirm canvas dimensions match what you want generated.

03

Open Create S-XL (Pro) from the PixelLab menu

The tool's name in the menu is Create S-L image (Pro) in some builds and Create S-XL image (Pro) in newer ones — the docs page title and the in-app label may differ.

04

Add reference images

Use the "Add reference image" button. You can attach up to 4 reference images to guide composition and content. Use these for: existing tiles you want incorporated, terrain palette references, art style anchors.

4 reference images: documented max
05

Optionally add a style image

Separate from reference images, the tool supports 1 style image that controls color palette, outline, detail level, and shading. Style options let you toggle which of these aspects to copy. Useful for matching an existing game's visual identity.

06

Write a strong description

Be explicit about perspective, scale, and content — the docs note that "the description should describe what's in the middle of the selected area." For maps, lead with viewpoint: "top-down 32x32 tile pixel art map: grassland with stone paths, a small lake, scattered trees, rocky outcrops".

07

Set seed and generate

Set a known seed if you want reproducibility for iteration. Click Generate. At 512×512 you receive 1 frame back (smaller canvases produce multi-frame variation grids — see the docs' frame output table).

08

Iterate via inpainting if needed

Rather than re-rolling at 40 generations a pop, fix specific regions with Workflow 08 (Inpaint v3). One full S-XL generation followed by 2-3 small inpaints is typically cheaper than multiple full re-rolls.

Caveats & uncertainties
  • 40 generations per attempt is a meaningful cost — be deliberate about your prompt and reference images before clicking Generate.
  • Single-pass generation means the model decides composition holistically; if you want fine control over specific regions, S-XL alone may not suffice and you will need inpaint follow-ups.
  • For tile-based Unity maps where each tile must be reusable and grid-aligned, dedicated map/tileset tools may still serve better than S-XL.
Workflow 05

External AI → Unzoom Pixel Art

Generate stylised pixel art externally in Gemini or ChatGPT at large resolution, then use Pixel Lab's Unzoom tool to detect the apparent pixel grid and rescale to true native pixel art.

External tool
Gemini / ChatGPT / DALL-E
Pixel Lab tool
Unzoom Pixel Art
Input range
256-2048px
Best for
Backgrounds, scenes
Procedure
01

Generate stylized pixel art externally

In Gemini or ChatGPT, request an image at 1024×1024 with explicit pixel art language. Be specific about the apparent pixel scale and palette: "top-down pixel art map, large 8px chunky pixels, 16-color limited palette, grassland with stone paths and trees, retro RPG style". The output will visually look like pixel art but is actually a high-resolution raster.

02

Save the output as PNG

Right-click → Save, or use the platform's download function. PNG preserves the apparent hard edges better than JPEG (which would smear pixel boundaries with compression artifacts).

03

Open the PNG in Pixelorama

File → Open, or drag and drop. In the import dialog, choose "New project" if prompted. Confirm the canvas dimensions match the source (e.g., 1024×1024).

04

Open Unzoom Pixel Art

From the PixelLab menu, find Unzoom Pixel Art. Per docs, this tool "automatically detects and removes zoom/upscaling from pixel art images. It analyzes the image to find the original pixel size and scales it down accordingly."

Min 256×256 / Max 2048×2048 Aseprite + Pixelorama only
05

Configure Color Reduction

Three modes available:
Auto — tool decides palette size (start here for first run).
None — keep all colors (rarely useful; AI output has thousands).
Custom — specify 2-256 colors. For retro feel, try 16-32; for richer pixel art, 48-64.

06

Run and inspect

The tool detects the upscale factor and outputs a downsized native-resolution image. If the AI's apparent pixels were ~8px, your 1024×1024 will become ~128×128. Inspect for irregular pixels — areas where the AI's grid drifted will show inconsistent block sizes.

07

Manual cleanup

Expect to clean: anti-aliased edges, irregularly-sized "pixels," color stragglers, broken outlines. Use the pencil and color-picker tools. Budget 10-30 minutes of cleanup per map for usable results.

08

Optional: scale up for use as art asset

If you want a larger asset for actual game use, scale the cleaned-up native pixel art back up using Nearest Neighbor interpolation (preserves crisp edges). Find this under Effects → Scale Image (the path is Effects, not Image, in the Pixel Lab fork).

Caveats & uncertainties
  • AI pseudo-pixel-art often has a drifting grid — pixels shift between, say, 7 and 9 px wide across the image. Unzoom's automatic detection picks the best average and leaves residual artifacts.
  • This workflow is poorly suited to tile-based Unity maps — the output is a single image, not a tileset of reusable cells. Best for splash screens, cutscenes, character portraits, illustrations.
  • External AI generators are non-deterministic and don't honor pixel-art constraints (palette discipline, clean lines, consistent dithering). Even after Unzoom + cleanup, the result may feel "off" compared to hand-crafted pixel art.
Workflow 06

External AI → Image to Pixel Art

A variant of Workflow 05 for source images that aren't already pixel-art-styled. Pixel Lab's Image-to-Pixel-Art tool stylizes any image into pixel art via an AI model, rather than detecting an existing pixel grid.

External tool
Any image source
Pixel Lab tool
Image to Pixel Art
Max output
200 / 256 / 320
Tier
Free / T1 / T2+
Procedure
01

Generate or source any image

Unlike Workflow 05, this tool doesn't require the source to look like pixel art. Use a regular illustration from Gemini/ChatGPT, a photo, concept art — anything. The model will reinterpret it as pixel art.

02

Import into Pixelorama

File → Open or drag and drop. The source image dimensions don't strictly matter since the tool will resize internally — but cleaner inputs (already cropped to the subject, balanced lighting) yield cleaner pixel art outputs.

03

Open Image to Pixel Art

From the PixelLab menu. Per docs: "the image is downsized 4 times; if you select a size larger than that, the image is automatically resized in the background to your requested image size."

04

Set the output size

Bound by your tier:

Free tier: 200×200 Tier 1: 256×256 Tier 2+: 320×320
05

Optionally set a Target Palette

The tool supports Target Palette — constrain the output to a specific color palette. For consistent style across multiple converted assets, use the same palette every run. Look for "Target Palette" in the tool options panel.

06

Configure Guidance weight and seed

Guidance weight controls how strictly the output matches the source. Higher = more faithful. Seed for reproducibility. Set both, then generate.

07

Choose Output method

Per docs, the tool has an Output method option (e.g., New frame, Modify current layer). Choose "New frame" to keep the source intact for comparison, or "Modify current layer" if you want to overwrite.

08

Generate and review

Output is true pixel art at the dimensions you specified. Unlike Unzoom, this is generative — it interprets the source rather than detecting an existing pixel grid. Better for source images that aren't already styled as pixel art.

Caveats & uncertainties
  • Output capped at 320×320 (Tier 2+) — substantially smaller than your 512×512 target. Not a single-pass solution for full-canvas maps; better suited for individual sprites, icons, characters, items.
  • Better for character/icon-style sources than for sprawling map scenes — diffusion-based stylization works best on focused subjects.
  • Distinct from Unzoom: this tool generates new pixel art interpretations. Unzoom detects existing pixel grids in pre-pixel-art-styled images. Pick the right tool for your source.
Workflow 08

Inpaint v3 with intentional masking

Targeted local edits without disturbing surrounding tiles. The 256×256 selection defines what the model can see; the painted Inpainting layer defines what gets replaced. Mastering this distinction is the key to surgical edits.

Tool
Inpaint (v3)
Selection size
up to 256×256
Cost
20 generations*
Tier
All tiers
Procedure
01

Open your existing map

Open the project containing the area you want to fix. Identify the specific pixels that need regenerating (e.g., "this pond should be grass"; "this castle wall is broken").

02

Open Inpaint v3 from the PixelLab menu

Per docs: "When you select Inpaint from the PixelLab menu, a new layer called Inpainting will be added to your image. Drawing on this layer specifies which parts of your image the model is allowed to modify."

Min 32×32 / Max 256×256 Inpainting layer auto-created
03

Make a selection that includes context

Critical conceptual point: the selection is the model's context window, not the area being replaced. Include the target zone plus surrounding healthy content so the model has style reference. For canvases > 256×256, the selection auto-snaps to 256×256.

04

Commit the selection

Press Enter or click the green checkmark in Left Tool Options. The corner handles disappear; you're left with a marching-ants outline. The selection is now locked and will persist when you switch to other tools.

05

Activate the Inpainting layer

In the Layers panel (timeline), click on the Inpainting layer to make it active. This is the layer your pencil strokes will land on — not your art layer. Switch to the pencil tool.

06

Paint BLACK over only the pixels to regenerate

Quote from Pixel Lab's Create Map guide: "if we want to have a consistent style and smooth transitions between tiles, it is important that we don't inpaint the whole selection area, as that leaves no reference for the model to use."

More paint = more freedom for the model. Less paint = stronger style anchoring. For "remove this pond," paint only the water tiles. For "redesign this whole region," paint nearly everything.

Color: black (#000000) "Paint in selection" must be checked
07

Write a focused description

The description should say what the painted area should become — not the whole scene. Per the docs: "the description should describe what's in the middle of the selected area." Example for the pond replacement: "detailed top-down grassland with small flowers".

08

Set output method and seed

Output method: "New frame" preserves the original; "Modify current layer" overwrites in place. Use New frame for first attempts (so you can compare), Modify current layer once you're confident in the prompt + mask combo.

Set a known Seed if you want reproducibility across attempts.

09

Generate

The model regenerates only the painted pixels, using the unpainted parts of the selection as style reference. Surrounding tiles stay untouched. Inspect the boundary between regenerated and original content — should blend seamlessly if mask + selection are well-chosen.

10

Iterate by adjusting the mask

If results are too divergent, paint less black (more anchor). If results are too constrained, paint more black (more freedom). Re-generate with the same seed to isolate the effect of the mask change.

Caveats & uncertainties
  • The 256×256 auto-snap behavior on selections is something you observed in your testing; it isn't explicitly documented as auto-snap. If you want truly smaller selections, the original Inpaint tool (under "Other") may behave differently — worth a single test.
  • Cost shown as 20 generations per docs; you observed 40 in-app. The in-app value is the source of truth — the docs page may be out of date.
  • Inpaint v3 options are: Description, No Background, Crop to Mask, Use Selection, Seed. Pixpatch v2 (also under Inpaint) offers more control — camera view, palette, guidance weight, projection — at the cost of a smaller 180×180 max.
